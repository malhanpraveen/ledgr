# CSV & PDF Import Design

Date: 2026-06-25

## Overview

Two new import paths in Settings: CSV (manual bulk entry, round-trip with export) and PDF statements (Gemini AI extraction via Vercel serverless function). Export CSV also updated to include `DueDay` column.

---

## CSV Format

Header (export and import share identical format):

```
Month,Label,Category,Amount,Recurring,DueDay
```

| Column | Type | Notes |
|---|---|---|
| Month | `YYYY-MM` | e.g. `2026-06` |
| Label | string | quoted if contains comma or newline |
| Category | string | must match existing or auto-created as custom |
| Amount | decimal | e.g. `15.99` |
| Recurring | `true`/`false` | |
| DueDay | integer 1–31 or empty | optional |

Example rows:
```
2026-06,Netflix,Entertainment,15.99,true,5
2026-06,Rent,Other,2000.00,false,
```

## CSV Import (`src/utils/importCsv.ts`)

Pure function — no Firestore, no side effects.

**Input:** raw CSV string  
**Output:**
```typescript
{
  expenses: Omit<Expense, 'id' | 'recurringSourceId'>[],
  newCategories: string[],   // categories not in existing set
  errors: string[],          // row-level parse errors (skipped rows)
}
```

**Rules:**
- Skip header row
- Skip blank rows
- Invalid rows (bad amount, bad month format, missing required fields) → add to `errors`, skip row
- Unknown category → add to `newCategories`, still include row
- `dueDay`: parse integer; empty/missing → `undefined`
- `isRecurring`: `"true"` (case-insensitive) → `true`, else `false`
- No duplicate detection — always import as new entries
- IDs generated by caller via `generateId()`

**SettingsView wires:**
1. `getDocs` existing categories
2. Call `parseCsv(text)`
3. Auto-create `newCategories` via `addCategory`
4. `writeBatch` all parsed expenses with fresh IDs
5. Show result: `✓ Imported N expenses (M new categories created).` or errors

## PDF Import (`api/extract-statement.ts`)

Vercel serverless function. API key never leaves server.

**Env var:** `GEMINI_API_KEY` (set in Vercel dashboard + local `.env`, never committed)

**Request:** `POST /api/extract-statement` with `multipart/form-data`, field `file` = PDF blob

**Flow:**
1. Upload PDF blob to Gemini Files API (`ai.files.upload`)
2. Call `ai.models.generateContent` with `gemini-2.0-flash-latest`, `responseMimeType: 'application/json'`, `responseJsonSchema`
3. Delete uploaded file (`ai.files.delete`)
4. Return extracted JSON

**Response schema:**
```typescript
{
  label: string,         // e.g. "Chase Sapphire Statement"
  amount: number,        // total amount due
  month: string,         // "YYYY-MM"
  dueDay: number | null, // day of month, null if not found
  isRecurring: boolean,
  category: string       // "Credit Card" | "Mortgage" | "Car" | "Other"
}
```

**Prompt to Gemini:**
> Extract billing details from this financial statement. For category, use one of: Credit Card, Mortgage, Car, Other. Set isRecurring to true if this is a regular monthly bill. Return total amount due (not minimum payment).

**Error handling:**
- Gemini API error → 500 with `{ error: message }`
- Invalid PDF → 400
- Missing env var → 500

## Export CSV change (`src/utils/exportCsv.ts`)

`buildCsvString` header changes from:
```
Month,Label,Category,Amount,Recurring
```
to:
```
Month,Label,Category,Amount,Recurring,DueDay
```

Each row appends `${e.dueDay ?? ''}`.
